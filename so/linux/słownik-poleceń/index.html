】Linuxowy słownik poleceń【

  • help - skrócona pomoc konkretnego polecenia
  • man "polecenie" - szczegółowe informacje na temat danej komendy
  • ls - wyświetla listę plików i katalogów znajdujących się w bieżącym katalogu lub w określonym miejscu w systemie plików
  • info - wypisuje informacje na temat wskazanej komendy
  • apropos - info o poleceniach związanych z danym tematem
  • what is - skrótowa forma co dane polecenie wykonuje
  • where is - podaje lokalizację fizyczną pliku polecenia oraz lokalizację strony manuala
  • cat - kompresuje i przygotowywuje pliki do standardowego wyjścia (sekcja 1)
  • useradd - tworzy nowego użytkownika, zmienia informacje o istniejącym użytkowniku
  • vim - edytor tekstu wiersza poleceń (sekcja 1)
  • less - opposite for more (sekcja 1)
  • more - przeglądanie plików tekstowych
  • tar - służy do umieszczania grupy plików w jednym zbiorczym pliku (czyli do tzw. archiwum), który następnie może zostać skompresowany, np. przy użyciu gzip (sekcja 1)
  • chown - zmienia właściciela pliku i grupę (sekckja 1)
  • chmod - zmiana uprawnień do plików (sekcja 1)
  • mkdir - tworzenie katalogów (sekcja 1)
  • rmdir - usuwanie katalogów
    • rm nazwa - usunięcie plików
    • rm -r nazwa -usunięcie katalogu z całą zawartością
  • ip adress - pomaga w wyświetlaniu interfejsów i adresów IP hosta aplikacji
  • cd
    • cd/ - przejście do katalogu głównego
    • cd /home - przejście do katalogu home
    • cd lub cd~ - przejście do katalogu domowego
  • touch - zakładanie prostego pliku lub zmiany daty jego modyfikacji
  • cp - kopiowanie plików i katalogów
    • cp lista.txt - kopiowanie pliku lista.txt pod nazwą kopia_lista.txt w bieżącym katalogu
  • mr:
    • mr lista.txt lista2.txt - zmiana nazwy pliku lista.txt na lista2.txt
    • mr lista2.txt k1 - przeniesienie pliku lista2.txt z katalogu bierzącego do katalogu k1
  • su root - przełącza bierzącego użytkownika na użytkownika root
  • su - użytkownik zastępczy
  • sudo - pozwala uruchomić aplikację jako inny użytkownik systemu
  • adduser - tworzenie w części skonfigurowanego konta użytkownika, konfiguracja opiera się o zapis w pliku /etc/adduser.conf
  • useradd - tworzenie konta użytkownika
  • chfn - zmiana danych konta
  • password - zmiana hasła bieżącego użytkownika
  • addgroup - tworzenie grupy
  • usermod -g "nazwa grupy domyślnej" "nazwa użytkownika"- zmiana grupy domyślnej
  • usermod -aG "nazwa grupy" "nazwa użytkownika"
  • grep "nazwa grupy" /etc/group - wyświetla użytkowników w grupie
  • deluser - usuwa użytkownika
  • delgroup - usuwa grupę
  • usermod -opcja "nazwa konta"

      opcje usermod:

    • -u - zmiana numeru UID
    • -g - zmiana grupy głównej
    • -G - dodanie lub usunęcie z dodatkoowej grupy
    • -d - zmiana ścieżki dostępu do katalogu domowego
    • -s - zmiana bieżącego shella
    • -i - zmiana loginu
    • -c - zmiana opisu konta
    • -f - zablokowanie konta (liczba dni do wygaśnięcia hasła)
    • -e - data utraty ważności konta (MM/DD/YY)
    • -L - zablokowanie hasła
    • -U - odblokowanie hasła
  • groupmood -opcja "nazwa grupy"

      opcje groupmod:

    • -g - zmiana numeru gid
    • -n - zmiana nazwy grupy
  • -id "nazwa konta - sprawdzanie przynależności grupowej"
  • groups "nazwa konta" - sprawdzanie przynależności grupowej
